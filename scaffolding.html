全局定义CSS 重置，排版，链接

1. 定义 HTML5 文档类型声明

Bootstrap 使用 HTML5 和 CSS3 技术所以需要 <!DOCTYPE html> 声明，确保在每个应用 Bootstrap 框架的页面都有这个声明。

<!DOCTYPE html>
<html lang="en">
  ...
</html>

排版及链接

通过修改 scaffolding.less 文件,可以修改布局参数，链接参数，默认的一些定义如下：

  • 移除<html>的外边距
  • 在 body上 background-color: white;
  • 设置 @baseFontFamily , @baseFontSize , and @baseLineHeight 变量作为我们排版的基础变量
  • 通过 @linkColor 变量设置全局链接的颜色，并且链接只有在 :hover 状态下显示下划线

通过Normalize.css重置浏览器默认样式

从 Bootstrap 2开始, Normalize.css 取代了传统的CSS样式表, 这是 Nicolas Gallagher 的项目， HTML5 Boilerplate 同样也是他的项目之一。

新的样式表保存于 reset.less 文件,移除一些元素后，它更加简洁并准确。

使用百分比计算定义宽度，而不是像素

流动网格系统使用百分比为列定义宽度而不是以往的固定像素定义宽度。确保适当的比例以适应一些关键的分辨率和设备。

流动行

可流动行与非流动行的差别仅仅是.row 与 .row-fluid.当然，流动列与非流动列之间的区别也是一样的，这是最简单直接的方法。

Markup

<div class="row-fluid">
  <div class="span4">...</div>
  <div class="span8">...</div>
</div>

流动嵌套

嵌套一个流动网格有一点不一样的地方: 嵌套在其中的列其和无需等于其父列。换句话说，嵌套在其中的列会重置其百分比宽度因为他们会占用100%的父列。

响应式设计 使用Media queries技术以创建应用于各种设备的响应式设计

为什么要使用Media queries

Media queries技术可以使CSS基于一些条件改变CSS定义，例如比例，宽度，设备类型等等。不过大部分会基于两个变量改变， min-width 和 max-width。

  • 重定义网格系统中列的宽度
  • 使网页元素堆叠在一起而不是任其浮动
  • 调整标题及文本的大小改变以更适应设备分辨率

谨慎的使用Media queries技术并且将其作为一个开端，为移动平台客户服务的开端。而对于大型项目，我们建议您为移动平台的用户考虑应用更专业的前端框架。

Bootstrap所支持的设备

Bootstrap支持如下5种类型的设备

类型 布局宽度 列宽度 缝隙宽度
智能手机 480px and below Fluid columns, no fixed widths
介于智能手机与平板电脑之间 767px and below Fluid columns, no fixed widths
平板电脑 768px and above 42px 20px
默认 980px and up 60px 20px
高分辨率显示器 1200px and up 70px 30px

加入必要的 meta 标签

为确保页面具有适当的可响应性，在 meta 标签内加入 viewport 属性。

<meta name="viewport" content="width=device-width, initial-scale=1.0">

如何使用 media queries 技术

Bootstrap并不会自动的应用 media queries 技术，不过应用他们确实十分的简单，只需三个步骤即可添加响应式特性：

  1. 使用编译过的bootstrap-responsive.css文件
  2. 添加@import "responsive.less"并重编译 Bootstrap
  3. 定义并重编译responsive.less文件

为什么不直接修改它呢？并不是所有的内容都需要应用 media queries 技术。这样做更易于移除 media queries 特性。

  // Landscape phones and down
  @media (max-width: 480px) { ... }

  // Landscape phone to portrait tablet
  @media (max-width: 768px) { ... }

  // Portrait tablet to landscape and desktop
  @media (min-width: 768px) and (max-width: 980px) { ... }

  // Large desktop
  @media (min-width: 1200px) { .. }

应用于的响应式设计的CSS类

是什么?

为了更快的开发移动设备上的应用，使用这些CSS类控制内容在不同设备上显示与否。

怎么使用呢？

使用这些CSS类，可以利用基本的内容来创建完全不同的版本。

例如,可以显示在手机上显示<select>元素作为导航栏，但是在显示器上确不显示它。
